# SHOT v2 — 종합 테니스 AI 플랫폼 설계 스펙

> 작성일: 2026-03-20
> 상태: Approved

---

## 1. 제품 비전

**"핸드폰 하나로 프로 수준의 테니스 분석을"**

SHOT은 삼각대에 고정된 핸드폰 카메라 하나로 테니스 경기와 연습을 자동 분석하는 Android 앱이다. 코트 라인 인식 + 공 추적 기술을 기반으로 인/아웃 판정, 서브 스피드 측정, 하이라이트 자동 생성, 샷 분포 분석 등을 제공한다.

### 핵심 가치 제안

- **동호인을 위한 호크아이**: 수억 원짜리 프로 장비의 핵심 기능을 핸드폰으로
- **객관적 데이터**: "느낌"이 아닌 숫자로 실력을 확인
- **바이럴 콘텐츠**: 자동 생성된 숏폼 클립으로 SNS 공유 → 앱이 알아서 퍼짐

## 2. 타겟 사용자

### Primary: 동호회 동호인 (한국 ~150만명)

- 주 1~3회 플레이
- 주말 복식/단식 경기 중심
- 네이버 카페/카톡 동호회 활동
- 경기 판정 시비, 실력 성장 확인에 대한 니즈
- 지불 의향: 월 3,000~10,000원

### Secondary (확장 단계)

- 레슨 프로/코치 (학생 분석 도구)
- 입문자/학원생 (연습 기록)

## 3. 설계 제약 조건

### 3-1. 촬영 조건 (강제)

| 항목 | 제약 |
|------|------|
| 핸드폰 위치 | 베이스라인 뒤 중앙, 1~2m 후방 |
| 높이 | 1~1.5m (삼각대) |
| 고정 | 삼각대 필수 (손 촬영 불가) |
| 방향 | 코트 정면 (횡단 촬영 불가) |

앱 시작 시 설치 가이드를 표시하고, 코트 인식 후 앵글이 부적절하면 경고를 표시한다. 이 제약은 코트 인식 정확도, 공 추적 정확도, 속도 계산 정확도를 보장하기 위한 것이다.

### 3-2. 기술 스택

| 항목 | 기술 |
|------|------|
| 플랫폼 | Android (minSdk 26) |
| 언어 | Kotlin (Android), Python (ML) |
| UI | Jetpack Compose |
| DI | Hilt |
| 카메라 | CameraX |
| ML 추론 | ONNX Runtime (Android) |
| 모델 학습 | PyTorch + Google Colab |
| 백본 | MobileNetV3-Small (코트), TrackNet 변형 (공) |
| 백엔드 | 미정 (Phase 3에서 결정) |

### 3-3. 성능 목표

| 항목 | 목표 |
|------|------|
| 코트 인식 | 키포인트 오차 < 2px (256×256 기준) |
| 공 추적 | 프레임당 공 위치 검출률 > 85% |
| 인/아웃 판정 | 정확도 > 90% (삼각대 고정 조건) |
| 서브 스피드 | 오차 ±10% 이내 (80~180km/h 범위). 실측 캘리브레이션 최소 3회(스피드건 비교 50서브+) |
| 추론 속도 | 코트: <15ms, 공: <25ms, 후처리: <10ms (총 <50ms). 코트 인식은 매 10프레임마다 실행 (코트는 고정이므로) |

## 4. 기능 설계

### 4-1. 기능 계층 구조

```
[기반 기술]
├── 코트 라인 인식 (8 keypoints → homography → 16 points)
└── 공 추적 (TrackNet: 연속 프레임에서 공 위치 검출)
    ↓
[핵심 기능]
├── 인/아웃 판정: 공 착지점 + 코트 좌표 → IN/OUT 판정
├── 서브 스피드 측정: 프레임 간 공 이동 거리 / 시간 → km/h
├── 영상 녹화: 삼각대 고정 경기/연습 영상 저장
└── 공 궤적 오버레이: 영상 위 공 이동 경로를 네온 라인으로 표시
    ↓
[분석 기능]
├── 샷 분포 히트맵: 코트를 4~6분할 → 구역별 착지 빈도
├── 서브 챌린지: 타겟 존에 서브 넣기 미니게임 + 점수
├── 경기/연습 리포트: 서브 속도 평균/최고, 인 비율, 샷 분포
└── 내 서브 vs 프로 비교: 착지점 분포를 프로 데이터와 대비
    ↓
[플랫폼 기능]
├── 숏폼 하이라이트 자동 생성: 랠리/에이스 구간 자동 클립 + 궤적 오버레이
├── 서브 스피드 전국 랭킹: 주간/월간, 지역별
├── 사용자 계정 + 데이터 동기화: 기록 보관, 성장 그래프
└── 프리미엄 구독: 과금 체계
```

### 4-2. 기능별 상세

#### A. 인/아웃 판정

**작동 방식:**
1. 공 추적 모델이 매 프레임에서 공 위치를 검출
2. 공의 y좌표 변화(하강→상승)로 바운스 순간 감지
3. 바운스 지점을 코트 좌표계로 변환 (homography)
4. ITF 코트 규격과 비교하여 IN/OUT 판정
5. 화면에 결과 표시 + 선택적 음성/진동 피드백

**판정 기준:**
- 공의 바운스 지점이 라인 포함 코트 영역 내 → IN
- 라인 밖 → OUT
- 판정 불가 (공이 가려짐, 추적 실패 등) → UNKNOWN (판정 안 함)

**정확도 보장 전략:**
- 판정 신뢰도 점수 함께 표시 (높음/중간/낮음)
- 낮은 신뢰도일 때 "판정 불확실" 표시 (잘못된 판정보다 나음)
- 리플레이에서 바운스 지점 시각화 → 사용자가 직접 확인 가능

#### B. 서브 스피드 측정

**작동 방식:**
1. 서브 모션 감지: 공이 코트 상단에서 하단으로 빠르게 이동
2. 연속 프레임에서 공 위치 추적 (최소 3프레임)
3. 코트 좌표계에서 실제 거리(미터) 계산
4. 프레임 간 시간 (1/30초 or 1/60초)으로 나눠 속도 산출
5. km/h로 변환하여 표시

**보정:**
- 카메라 프레임레이트에 따른 보정
- 공의 3D 궤적 (포물선) 고려 → 2D 투영 속도보다 실제 속도가 빠름
- 보정 계수 적용 (실측 대비 캘리브레이션)

#### C. 숏폼 하이라이트 자동 생성

**작동 방식:**
1. 경기 녹화 중 "이벤트" 자동 감지:
   - 에이스 (서브 → 바운스 → 상대 터치 없음)
   - 긴 랠리 (N회 이상 공이 네트 통과)
   - 빠른 서브 (일정 속도 이상)
2. 이벤트 전후 3~5초를 잘라 숏폼 클립 생성
3. 공 궤적 슬로모션 오버레이 자동 적용
4. 서브 스피드, 인/아웃 결과 자막 삽입
5. 15~30초 하이라이트 영상 자동 편집

**과금:**
- 무료: 숏폼 생성 가능, SHOT 워터마크 포함
- 유료: 워터마크 제거 + HD 품질 + 커스텀 템플릿

#### D. 서브 챌린지 모드

**작동 방식:**
1. 코트 인식 후 서비스 박스에 타겟 존 AR 오버레이 표시
   - 예: 와이드/바디/T 존 3분할 또는 9분할 그리드
2. 사용자가 서브 → 공 착지점 자동 감지
3. 타겟 존 적중 여부 판정 + 점수 부여
4. 세션 종료 후 결과 카드: "10/15 적중, 정확도 67%, 평균 속도 108km/h"

**게이미피케이션:**
- 연속 적중 콤보 보너스
- 일일/주간 챌린지 (예: "오늘의 미션: T존에 5개 연속")
- 결과 카드 SNS 공유

#### E. 샷 분포 히트맵

**작동 방식:**
1. 경기/연습 중 모든 공 착지점을 코트 좌표로 기록
2. 코트를 4분할(듀스/에드 × 서비스/베이스) 또는 6분할로 구역 나눔
3. 구역별 빈도수 + 비율 히트맵으로 시각화
4. 서브/포핸드/백핸드 별도 필터링 (Phase 3에서 샷 분류 추가 시)

**표시:**
- 색상 그라데이션 (파랑→빨강)으로 빈도 시각화
- 구역별 숫자 + 비율 표시
- 경기별 / 기간별 비교 가능

#### F. 서브 스피드 전국 랭킹

**작동 방식:**
1. 서브 스피드 측정 시 기록을 서버에 업로드
2. 주간/월간 랭킹 집계
3. 전체 / 지역별 / 동호회별 순위 표시
4. 주간 리셋으로 지속적 참여 유도

**부정 방지:**
- 코트 인식 성공 상태에서만 기록 인정
- 비정상적 속도 (200km/h+) 자동 필터링
- 영상 첨부 의무화 또는 랜덤 검증

#### G. 내 서브 vs 프로 비교

**작동 방식:**
1. 사용자의 서브 착지점 분포 데이터 축적
2. 프로 선수 서브 데이터 (공개 자료: IBM 스탯, ATP/WTA 공식 데이터 활용)
3. 오버레이 비교: "당신의 듀스 코트 와이드 서브 비율: 43% / 나달: 38%"
4. 종합 유사도 점수: "당신의 서브 패턴은 나달과 73% 유사합니다"

#### H. 공 궤적 슬로모션 오버레이

**작동 방식:**
1. 녹화된 영상에서 공 위치 데이터 추출 (후처리)
2. 공의 이동 경로를 네온 컬러 라인으로 프레임 위에 렌더링
3. 바운스 지점에 임팩트 이펙트 표시
4. 0.25x ~ 0.5x 슬로모션 적용
5. 숏폼 클립에 자동 포함

### 프리미엄 + 워터마크 하이브리드 (모델 C)

| 구분 | 무료 | 프리미엄 (월 4,900원) |
|------|------|----------------------|
| 코트 인식 | ✅ | ✅ |
| 인/아웃 판정 | ✅ 무제한 | ✅ |
| 서브 스피드 표시 | ✅ | ✅ |
| 영상 녹화 | ✅ | ✅ |
| 숏폼 생성 | ✅ 워터마크 포함 | ✅ 워터마크 제거 + HD |
| 공 궤적 오버레이 | ✅ | ✅ |
| 샷 분포 히트맵 | 최근 3회분 | ✅ 무제한 |
| 서브 챌린지 | 일 1회 | ✅ 무제한 |
| 서브/샷 리포트 | 최근 5회분 | ✅ 무제한 + 성장 그래프 |
| 프로 비교 | ❌ | ✅ |
| 전국 랭킹 | 조회만 | ✅ 등록 + 순위 |

### 과금 전략

- **핵심 기능(인/아웃, 스피드)은 완전 무료** → 다운로드 장벽 제거
- **숏폼 워터마크** → 바이럴 + 유료 전환 유도 (워터마크 = 무료 광고)
- **데이터 축적 lock-in** → 기록이 쌓일수록 떠나기 어려움
- **제한 횟수** → 맛보기 후 "더 쓰고 싶으면 구독"

## 6. 기술 아키텍처

### 6-1. ML 파이프라인

```
카메라 프레임 (30fps)
    ↓
[코트 인식 모델] MobileNetV3-Small + Heatmap Decoder
    → 8 keypoints (니어코트) → Homography → 16 points (전체 코트)
    ↓
[공 추적 모델] TrackNet 변형 (연속 3프레임 입력)
    → 공 위치 (x, y) + confidence
    ↓
[후처리]
    ├── 바운스 감지 (y좌표 변화 패턴)
    ├── 착지점 → 코트 좌표 변환 (homography)
    ├── 속도 계산 (프레임 간 거리 / 시간)
    └── 인/아웃 판정 (코트 영역 내/외)
    ↓
[렌더링]
    ├── 코트 라인 오버레이
    ├── 공 궤적 표시
    ├── 인/아웃 결과 표시
    └── 서브 스피드 표시
```

### 6-2. Android 모듈 구조

```
app/                    # 메인 앱 (UI, ViewModel, Navigation)
├── camera/             # CameraX 관리 + 영상 녹화
├── court-detection/    # 코트 인식 ONNX 모델 추론
├── ball-tracking/      # 공 추적 모델 추론 (신규)
├── court-model/        # 호모그래피, 투영, 검증, 안정화
├── analysis/           # 인/아웃 판정, 속도 계산, 샷 분포 (신규)
├── challenge/          # 서브 챌린지 모드 (신규)
├── highlight/          # 숏폼 생성, 궤적 오버레이 (신규)
├── core/               # 데이터 모델, ITF 스펙
└── sync/               # 백엔드 동기화 (Phase 3, 신규)
```

### 6-3. 데이터 흐름

```
[실시간 모드]
카메라 → 프레임 → 코트 인식 + 공 추적 → 분석 → UI 업데이트
                                            ↓
                                      로컬 DB 저장 (착지점, 속도 등)

[후처리 모드 (숏폼 생성)]
저장된 영상 → 프레임별 공 추적 → 이벤트 감지 → 클립 생성 → 오버레이 렌더링

[동기화 모드 (Phase 3)]
로컬 DB → 서버 업로드 → 랭킹 집계 → 클라이언트 동기화
```

## 7. 로드맵

### Phase 1a: 공 추적 데이터 + 모델 (현재 ~ 3주)

| 작업 | 상세 | 산출물 |
|------|------|--------|
| TrackNet 학습 데이터 수집 | 삼각대 고정 촬영 영상 수집 (최소 10경기, 30fps) | 원본 영상 |
| 공 위치 라벨링 | 최소 5,000프레임 라벨링 (x, y, visibility flag). 라벨링 도구 제작 | annotations.json |
| TrackNet 모델 학습 | 입력: 3×360×640 (연속 3프레임), 출력: 히트맵 1×360×640. Colab 학습 | ball_tracking.onnx |
| 데이터 검수 | 학습 데이터 검수 완료 후 학습 진행 (검수 전 학습 금지) | 검수 보고서 |
| **마일스톤** | 테스트 영상에서 공 검출률 > 85% | |

### Phase 1b: Android 통합 + 판정 (3주 ~ 6주)

| 작업 | 상세 | 산출물 |

## 10. TrackNet 모델 상세

### 아키텍처
- **기반**: TrackNet v2 (연속 프레임 기반 공 추적)
- **입력**: 3프레임 연결 (t-2, t-1, t) → 9×360×640 (3프레임 × 3채널 RGB)
- **출력**: 히트맵 1×360×640 (공 위치 가우시안)
- **백본**: 경량화된 encoder-decoder (모바일 추론 고려)
- **Confidence**: 히트맵 최대값 > threshold → 공 검출, 미만 → 미검출

### 학습 데이터 요구사항
- **최소 데이터**: 5,000 프레임 (10개 이상 경기/연습 영상에서 추출)
- **어노테이션 포맷**: 프레임별 (x, y, visibility). visibility: 0=미보임, 1=보임, 2=가려짐
- **환경 커버리지**: 실내 2종 이상, 실외 2종 이상, 주간+야간, 클레이+하드코트
- **검수 기준**: 라벨링 후 랜덤 100프레임 샘플 검수 → 오류율 < 5%이어야 학습 진행

### 바운스 감지 알고리즘
1. 연속 프레임에서 공 y좌표 추적
2. Kalman 필터로 노이즈 제거
3. y좌표 5프레임+ 하강 → 3프레임+ 상승 패턴 감지 = 바운스 후보
4. 바운스 시점의 공 위치를 homography로 코트 좌표 변환
5. 3프레임 이상 공이 미검출된 구간은 바운스 판정 보류
6. 선수 뒤 가려짐(visibility=2)은 선형 보간으로 위치 추정하되 판정 신뢰도를 "낮음"으로 표시

### 숏폼/하이라이트 생성
- **처리 방식**: 후처리 (실시간 아님). 경기 종료 후 백그라운드에서 처리
- **예상 처리 시간**: 1분 영상 당 약 2~3분 (중급 기기 기준)
- **비디오 인코딩**: Android MediaCodec API 사용 (FFmpeg 번들 불필요)
- **출력**: MP4, 720p, 15~30초 클립
