# Phase 2b: 단일프레임 볼 검출기 + 칼만 필터

> 작성일: 2026-03-21

## 배경

현재 TrackNet은 3프레임 입력 [1, 9, 128, 320] → 148ms 추론 → 최대 6fps.
공이 프레임 사이에 7m 이동하므로 추적 불가. 3프레임 시간적 의존성이 6fps에서 깨지고,
방송 영상(30fps) 학습 데이터와 도메인 갭도 큼.

## 변경 구조

```
[현재] 3프레임 → TrackNet 1.93M (148ms) → 위치         → 6fps
[변경] 1프레임 → 경량 검출기 ~200K (15-25ms) → 칼만 필터 → 30-50fps
```

## 새 모델 아키텍처

- **백본**: MobileNetV3-Small (ImageNet pretrained)
  - 코트 검출 모델과 동일 백본 — 디바이스에서 검증 완료
  - features 출력: [B, 576, 6, 6] (192÷32)
- **헤드**: 경량 디코더 (ConvTranspose2d × 3)
  - 576 → 128 → 64 → 32 → 1채널
  - 6×6 → 12×12 → 24×24 → 48×48 히트맵
- **입력**: [1, 3, 192, 192] 단일 RGB 프레임
- **출력**: [1, 1, 48, 48] 히트맵 (sigmoid)
- **파라미터**: ~200K (현재 1.93M의 1/10)
- **정규화**: ImageNet (mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225])

입력 해상도 192×192 선택 이유: 공은 원형이라 aspect ratio 왜곡에 둔감.
히트맵 argmax → 원본 좌표 복원 시 x/y 독립 스케일백이므로 최종 좌표에 영향 없음.
성능 부족 시 192×320(16:9)으로 확장.

## 칼만 필터 트래킹

검출기가 매 프레임 위치를 찾고, 칼만 필터가 시간적 연속성을 부여:

- **상태 벡터**: [x, y, vx, vy, ax, ay] (위치 + 속도 + 가속도)
- 검출 성공 시: 실측값으로 상태 보정 (measurement update)
- 검출 실패 시: 물리 기반 위치 예측 (predict only, 1-2프레임 허용)
- 이상치 거부: 예측 위치에서 너무 먼 검출은 무시 (gating)
- 30fps에서 대부분 프레임 검출 성공하므로 예측은 순간 갭 보충용

## 목표 스펙

| 지표 | 최소 | 목표 |
|------|------|------|
| FPS (FP32) | 25 | 35 |
| FPS (INT8 양자화) | 35 | 50 |
| FPS (LOCK + INT8) | 40 | 60 |
| 검출률 | 85% | 92% |
| 위치 오차 | < 20px | < 15px (1280×720) |
| 궤적 연속성 (칼만 보정 후) | 90% | 95%+ |
| False positive | < 5% | < 3% |

칼만 필터 예측: 30fps 기준 1프레임 미검출 시 오차 < 2px.
연속 3프레임 이상 미검출 시 예측 중단, 마커 숨김.

## 학습 데이터

- **기존**: TrackNet Dataset-001 (19,835 프레임, 방송 영상) → 단일프레임으로 변환
- **추가 수집**: SNTC YouTube 핸드폰 영상에서 프레임 추출 + 라벨링 (목표 30,000+)
- 기존 `label_ball.py` 라벨링 도구 활용
